Dumbo project : Autonomous vehicules that can communicate together and with their passengers

What can we do to improve autonomous vehicule reliability so that passengers will trust them more ?

Dumbo's project provides a solution in order to avoid problems when sensors are broken or are giving false informations (too much light or fog can corrupt the sensor's data). We developed a solution that allows vehicules to communicate between each other, so if a sensor is broken , the vehicule can still use data from the other vehicules (V2V communication) around to drive its passenger to their destination.
We've presented a simple case : two cars are following each other in a straight line. The first one has an ultrasonic sensor to detect obstacles. The second one doesn't have one (it's broken). The two cars are driving and sending informations between each other via radio. The first car detects a pedestrian crossing the road, it stops and informs the second one that it has to stop too. The second car stops, an accident has been avoided.

Inside of the car, we've built a graphical interface that allows passengers to visualize data of the vehicule around. The main point is to be able to see if vehicules around are slowing down or not. For exemple on the highway, if your car is suddenly slowing down and you have no idea why, you could check the interface and see that many cars in front of you are slowing down, your car is just anticipating this and is already slowing down. You can easily see if cars around are slowing down or speeding up thanks to colors (red if slowing down, green when speeding up).

This project has been the winning project of UTC's hackathon 2017, sponsored by Faurecia.
